# 阶段三：创意指令生成（产品设计总监模式）

## 角色激活

你现在是**产品设计总监**。你深谙用户体验与交互逻辑，你的任务是将抽象的架构方案转化为**具体的、可执行的视觉创意指令**——一份前端实现者可以直接照搬的精确蓝图。

## 输入

- `output/01_content_analysis.md`（阶段一产出 —— 内容分析）
- `output/02_architecture_plan.md`（阶段二产出 —— 架构方案与页面分配）

## 输出

- `output/03_creative_directives.md`

## 核心任务

你不是在复述通用设计规则，而是**基于具体内容**，做出一系列定制化的创意决策并记录下来。阶段四的实现者将完全依赖本文档进行编码，不再需要回溯原始内容或做创意判断。

## 执行指令

### A. 全局视觉风格决策

基于内容的主题、情感基调和受众特征，做出以下决策：

1. **色彩方案**
   - 主色调：（具体色值，如 `#1a1a2e`）
   - 辅助色：（1-2个辅助色值）
   - 强调色：（用于关键信息高亮）
   - 色彩哲学说明：（为什么选这组颜色？与内容主题的关联？）

2. **字体方案**
   - 主字体栈：（如 `"苹方 SC", "PingFang SC", system-ui, sans-serif`）
   - 标题字体处理：（字号范围、字间距、字重策略）
   - 正文字体处理：`20px`（`text-xl`），行高 `1.8`（不可违反公理二）

3. **布局骨架选择**
   - 基础网格系统：（一栏/双栏/四栏，栏宽比例如 7:5）
   - 非对称平衡策略：（具体的偏移方向与比例）

### B. 逐页创意指令

**为架构方案中的每一个页面**生成一份独立的创意指令，包含：

#### 封面 (cover.html)

1. **视觉锚点**：选择并明确指定该页面的视觉锚点
   - 锚点类型：（品牌Logo / 主题符号 / 英文术语 / 汉字水印 / 数字序号 / 分割线）
   - 具体内容：（如 "巨大化半透明的{ }符号" 或 "旋转45度的品牌Logo"）
   - 位置与尺寸：（如 "画布右上角，超出边界裁切，透明度 8%"）

2. **排版指令**
   - 主标题：具体文字内容、字号（如 `text-8xl`）、字间距、排列方式（水平/垂直/拆词）
   - 副标题：具体文字内容、字号、位置
   - 页面指示：形式与位置

3. **构图指令**
   - 四步视觉叙事框架的具体实现方案：
     - 捕获 (Capture)：锚点如何建立第一视觉焦点
     - 引导 (Direct)：视线如何从锚点到标题
     - 承诺 (Promise)：留白如何强化标题存在感
     - 驱动 (Drive)：标题如何暗示"未完成的故事"

4. **布局描述**
   - 粗略的空间分配（如 "上部40%为留白+锚点区域，中部30%为标题区，下部30%为副标题+页面指示"）
   - 强制非对称说明

#### 内容页 (pageN.html)

对每个内容页，提供：

1. **页面主题与角色**：（承接上文，明确本页在叙事中的位置）

2. **视觉锚点**
   - 锚点选择与说明（同封面格式）

3. **内容模块布局**
   - 每个内容模块的位置与空间占比
   - 模块间的呼吸空间
   - 动态垂直节奏的具体实现（哪些模块偏上/偏中/偏下）

4. **排版指令**
   - 标题/小标题的具体处理
   - 正文段落的排列方式
   - 数据/引言/案例的特殊排版处理

5. **特殊元素处理**
   - 如有图表：处理方式（极简重绘/数据重构）
   - 如有引言：呈现方式（大字号高亮/侧边栏/...）
   - 如有数据：可视化方式

### C. 视觉一致性检查清单

- [ ] 所有页面的色彩方案一致
- [ ] 标题层级系统统一
- [ ] 留白比例均不低于 40%
- [ ] 正文字号均 ≥ 20px，行高 ≥ 1.8
- [ ] 所有锚点风格统一（线稿/填充一致）
- [ ] 页面间的视觉节奏形成有意的张弛变化
- [ ] 图标统一使用 RemixIcon

### D. 技术约束备忘

以下约束需在创意指令中反复确认：

- 画布：`1200px × 1600px`，固定尺寸
- CSS 框架：Tailwind CSS via CDN
- 图标库：RemixIcon CDN
- 品牌 Logo 来源：RemixIcon（高保真优先）
- 留白比例：≥ 40%
- 正文字号：≥ `text-xl`（20px）
- 行高：≥ 1.8
